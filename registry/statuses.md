# Статусы документации Websource

Этот файл описывает статусы, которые используются в документации Websource.

Статус показывает текущее состояние элемента документации: класса, метода, функции, переменной, модуля, настройки или другого элемента API.

## Где используется статус

Статус указывается во frontmatter Markdown-файла.

Пример frontmatter:

~~~yaml
---
title: WSMailer
slug: wsmailer
doc-id: wsmailer
lang: ru
version: latest
section: api
type: classes
status: stable
description: Класс для отправки email сообщений через SMTP или mail().
introduced-in: 0.87
---
~~~

## Основные статусы

| Статус | Значение | Когда использовать |
|---|---|---|
| `stable` | Актуальный элемент | Элемент существует в текущей версии и рекомендован к использованию |
| `draft` | Черновик | Элемент ещё не утверждён и может измениться |
| `experimental` | Экспериментальный элемент | Элемент уже реализован, но его API может измениться |
| `deprecated` | Устаревший элемент | Элемент ещё может существовать в коде, но не рекомендуется для нового использования |
| `removed` | Удалённый элемент | Элемент больше не существует в текущей версии кода |
| `internal` | Внутренний элемент | Элемент используется внутри CMS и не предназначен для публичной документации |
| `planned` | Запланированный элемент | Элемент описывает будущую возможность, которая ещё не реализована |
| `unknown` | Статус неизвестен | Используется временно, если статус нельзя определить по коду или реестрам |

## Правила использования

### stable

Используется для актуальных элементов.

Такой элемент можно описывать как доступный и рекомендуемый к использованию.

Пример:

~~~yaml
status: stable
~~~

### draft

Используется для черновиков.

Такой элемент нельзя считать окончательной частью публичной документации.

Пример:

~~~yaml
status: draft
~~~

### experimental

Используется для экспериментальных возможностей.

Элемент может работать, но его API или поведение могут измениться в будущих версиях.

Пример:

~~~yaml
status: experimental
~~~

### deprecated

Используется для устаревших элементов.

Такой элемент нельзя рекомендовать для нового кода. Если есть замена, её нужно указать.

Рекомендуемые поля:

~~~yaml
status: deprecated
deprecated-in:
replacement:
~~~

Где:

- `deprecated-in` — версия, начиная с которой элемент считается устаревшим;
- `replacement` — элемент, который нужно использовать вместо устаревшего.

### removed

Используется для удалённых элементов.

Такой элемент нельзя описывать как доступный в текущей версии.

Рекомендуемые поля:

~~~yaml
status: removed
removed-in:
replacement:
~~~

Где:

- `removed-in` — версия, начиная с которой элемент удалён;
- `replacement` — элемент, который нужно использовать вместо удалённого, если такая замена есть.

### internal

Используется для внутренних элементов CMS.

Такие элементы можно описывать только если документация предназначена для разработчиков ядра Websource.

Пример:

~~~yaml
status: internal
~~~

### planned

Используется для запланированных возможностей.

Такой статус нельзя использовать для описания уже существующего API, если элемент ещё не реализован в коде.

Пример:

~~~yaml
status: planned
~~~

### unknown

Используется временно, если статус не удалось определить.

Элементы со статусом `unknown` нельзя публиковать как окончательную документацию без проверки.

Пример:

~~~yaml
status: unknown
~~~

## Связь с версиями

Статус элемента не заменяет версию документации.

Версия страницы указывается отдельно:

~~~yaml
version: latest
~~~

или:

~~~yaml
version: v1.0
~~~

Версия появления элемента указывается отдельно:

~~~yaml
introduced-in: 0.87
~~~

Если элемент устарел или удалён, можно использовать дополнительные поля:

~~~yaml
deprecated-in:
removed-in:
replacement:
~~~

## Главное правило

Если элемент отсутствует в текущем коде, его нельзя автоматически описывать как `stable`.

Сначала нужно определить:

1. элемент был удалён;
2. элемент переименован;
3. элемент находится в другой ветке;
4. элемент относится к старой версии документации;
5. элемент ещё не реализован.

Если это невозможно определить по коду или реестрам, статус должен быть `unknown`.
