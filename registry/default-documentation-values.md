# Значения по умолчанию

## Общие правила

Если значение невозможно подтвердить по коду, реестрам или существующей документации, его нельзя придумывать.

## Status

Если статус неизвестен:

поле status не выводить.

## introduced-in

Если версия появления неизвестна:

поле introduced-in не выводить.

## deprecated-in

Если версия устаревания неизвестна:

поле deprecated-in не выводить.

## removed-in

Если версия удаления неизвестна:

поле removed-in не выводить.

## replacement

Если замена неизвестна:

поле replacement не выводить.

## Description

Если официальное описание отсутствует:

разрешается сформировать техническое описание на основе подтверждённого поведения кода.

## Принцип

Документация должна содержать только подтверждённые данные.

Отсутствие значения предпочтительнее, чем неподтверждённое значение.
