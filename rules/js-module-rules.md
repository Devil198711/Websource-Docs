# Правила документирования JavaScript-модулей

Экспортируемые функции, классы и объекты считаются публичным API модуля, если не указано обратное.

Если модуль имеет простой способ использования и расширенный API, документация должна сначала описывать простой способ использования.

После этого может быть описан расширенный API.

Не документировать внутренние методы, которые не предназначены для прямого использования разработчиком.

Не описывать внутреннюю реализацию, если она не влияет на использование публичного API.
