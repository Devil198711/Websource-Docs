# {Название модуля}

Краткое описание модуля в 1–3 предложения.

## Назначение

Объяснить, какую задачу решает модуль.

Описывать пользовательские возможности, а не внутреннюю реализацию.

## Поддерживает

Список основных возможностей модуля.

Использовать только возможности, подтверждённые кодом.

## Быстрый старт

Минимальный рабочий пример использования.

## Подключение

Если для работы требуется импорт, подключение файла или инициализация, описать их здесь.

## HTML-структура

Описать обязательную HTML-разметку.

Указать:

- обязательные элементы;
- обязательные data-атрибуты;
- необязательные элементы;
- дополнительные data-атрибуты.

Использовать только структуру, подтверждённую кодом.

## Инициализация

Описание основного способа запуска модуля.

Пример:

js
const module = someModule(...)

## Параметры

Таблица параметров.

| Параметр | Тип | Описание |
|---|---|---|

Указывать только параметры, реально используемые модулем.

## Режимы работы

Использовать только если модуль поддерживает несколько режимов.

Для каждого режима описать:

- назначение;
- особенности;
- ограничения;
- пример использования.

## Публичный API

Документировать только публичный API модуля.

Если модуль экспортирует функции:

### functionName()

Описание.

Если модуль экспортирует классы:

### ClassName

Описание класса.

### methodName()

Описание метода.

Документировать только public API.

## События

Использовать только если модуль генерирует события.

Указать:

- название события;
- когда вызывается;
- доступные данные.

## Data-атрибуты

Использовать только если модуль работает через data-атрибуты.

Формат:

| Атрибут | Назначение |
|---|---|

## Ограничения и замечания

Только подтверждённые ограничения.

Не описывать внутренние алгоритмы работы.

## Что использовать

Использовать, если модуль поддерживает несколько способов работы.

Например:

- использовать режим A для ...
- использовать режим B для ...
