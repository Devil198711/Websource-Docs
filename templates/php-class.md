## Основная структура страницы

# {Название класса}

Краткое описание класса в 1–3 предложения.

## Поддерживает

Список основных возможностей класса.

Использовать только то, что подтверждено кодом.

## Быстрый старт

Минимальный рабочий пример использования класса.

Если безопасный пример нельзя составить по коду, раздел не заполнять без уточнения.

## Конфигурация

Использовать только если класс принимает конфигурацию, настройки или зависит от внешних параметров.

Для конфигурации использовать таблицу:

| Параметр | Тип | Описание |
|---|---|---|

## Инициализация

Описать конструктор класса.

Формат:

### __construct(...)

Описание.

Параметры:

| Параметр | Тип | Описание |
|---|---|---|

Пример:

```php
// пример создания объекта
```

## Основные сценарии использования

Использовать, если класс имеет несколько разных сценариев работы.

Например:

- отправка текстового письма;
- отправка HTML-письма;
- проверка соединения;
- получение состояния.

## Публичные методы

Документировать только публичные методы класса.

Private- и protected-методы не выносить в публичный API, если пользователь явно не просит внутреннюю документацию.

Формат метода:

### methodName(type $param = default) : returnType

Краткое описание метода.

Параметры:

| Параметр | Тип | Описание |
|---|---|---|

Возвращает:

| Тип | Описание |
|---|---|

Пример:

```php
// пример вызова метода
```

## Форматы данных

Использовать, если метод принимает массивы, конфигурацию или сложные структуры данных.

Например:

| Ключ | Тип | Описание |
|---|---|---|

## Исключения

Указать исключения, которые явно выбрасываются в коде.

Формат:

| Исключение | Когда возникает |
|---|---|

Не придумывать исключения, которых нет в коде.

## Ограничения и замечания

Список важных ограничений, подтверждённых кодом.

Например:

- работает только с одним получателем;
- требует валидный email;
- использует fallback на другой механизм;
- зависит от глобального объекта или константы.

## Что использовать

Использовать, если у класса есть несколько похожих методов и нужно объяснить выбор.

Например:

- используйте `sendText()`, если нужно отправить обычный текст;
- используйте `sendHtml()`, если нужно отправить HTML.
