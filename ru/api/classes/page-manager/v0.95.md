# PageManager

`PageManager` получает данные опубликованных страниц независимо от текущего HTTP-запроса.

## Поддерживает

- получение опубликованной страницы по URL
- проверку поддержки свободной ссылки
- работу с основной или переданной базой данных
- нормализацию начальных и конечных `/` в URL

## Быстрый старт

```php
$pageManager = new PageManager();
$page = $pageManager->getPublishedByUrl('contacts');

if ($page === null) {
    return;
}

$title = $page['title'];
$content = $page['content'];
```

## Получение страницы

### getPublishedByUrl(string $url, bool $freeLink = false): ?array

Возвращает опубликованную страницу по её URL.

```php
$page = (new PageManager())->getPublishedByUrl('contacts');
```

Начальные и конечные символы `/` удаляются, поэтому допустим и такой вызов:

```php
$page = (new PageManager())->getPublishedByUrl('/contacts/');
```

Если URL пустой или страница не найдена, метод возвращает `null`.

### Параметры

| Параметр | Тип | Описание |
|---|---|---|
| `$url` | `string` | URL страницы. |
| `$freeLink` | `bool` | Требовать у страницы непустое значение `free_link`. |

Метод возвращает данные записи страницы без дополнительного преобразования метаданных.

## Свободная ссылка

Чтобы получить только страницу с поддержкой свободной ссылки, передайте `true` вторым параметром:

```php
$page = (new PageManager())->getPublishedByUrl(
    'developer',
    true
);
```

Если подходящая страница отсутствует, метод вернёт `null`.

## База данных

### __construct(?Database $db = null)

Без аргументов менеджер использует основной объект базы данных CMS:

```php
$pageManager = new PageManager();
```

При необходимости можно передать другой экземпляр `Database`:

```php
$pageManager = new PageManager($database);
```

## Отличие от PageData

`PageManager` используется для поиска произвольной опубликованной страницы по URL.

`PageData` содержит данные страницы или записи, уже определённой CMS для текущего запроса.

## Важные замечания

- метод возвращает только страницы со статусом `publish`
- пустой URL возвращает `null`
- отсутствующая страница возвращает `null`
- метаданные возвращаются без дополнительного преобразования
- получение страницы по идентификатору класс не поддерживает
- для данных текущей страницы используйте `PageData`
