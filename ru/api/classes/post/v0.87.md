# Post

`Post` предоставляет статические методы для работы с записями разных типов.

## Поддерживает

- получение одной записи
- получение списков записей
- выбор только опубликованных записей
- фильтрацию и сортировку
- объединение нескольких типов записей
- подсчёт записей
- работу с метаданными и категориями
- добавление и обновление записей

## Быстрый старт

```php
$posts = Post::getPublishPost('news', 0, 10);

foreach ($posts as $post) {
    echo Convert::sanitizeHtml($post['title']);
}
```

Создавать экземпляр класса не нужно.

## Формат результата

Методы получения списков возвращают массив, индексы которого начинаются с `1`.

```php
$posts = Post::getPublishPost('news', 0, 10);

$firstPost = $posts[1] ?? null;
```

Поля со структурированными данными преобразуются в массивы:

- `meta`
- `category`
- `global_meta`
- `global_category`
- `comment`

Каждая запись также получает ключ `post_type` с названием её типа.

## Получение одной записи

### getPost($id, $type): array|null

Возвращает запись по идентификатору и типу.

```php
$post = Post::getPost(15, 'news');

if ($post) {
    echo Convert::sanitizeHtml($post['title']);
}
```

| Параметр | Описание |
|---|---|
| `$id` | Идентификатор записи. |
| `$type` | Системное имя типа записи. |

Результат дополнительно содержит `author_login` с логином автора.

Метод не проверяет статус и дату публикации записи.

## Получение списка записей

### getAllPost($type, $limit = '', $limit2 = '', $order = '`id`', $sort = 'DESC'): array

Возвращает записи указанного типа независимо от их статуса.

```php
$posts = Post::getAllPost('news', 0, 10);
```

| Параметр | Описание |
|---|---|
| `$type` | Системное имя типа записи. |
| `$limit` | Смещение или максимальное количество записей. |
| `$limit2` | Количество записей при использовании смещения. |
| `$order` | Поле сортировки. |
| `$sort` | Направление сортировки: `ASC` или `DESC`. |

Получение первых десяти записей:

```php
$posts = Post::getAllPost('news', 10);
```

Получение десяти записей после первых двадцати:

```php
$posts = Post::getAllPost('news', 20, 10);
```

Сортировка по заголовку:

```php
$posts = Post::getAllPost('news', 0, 10, '`title`', 'ASC');
```

### getPublishPost($type, $limit = '', $limit2 = '', $order = '`id`', $sort = 'DESC'): array

Возвращает опубликованные записи, дата публикации которых уже наступила.

```php
$posts = Post::getPublishPost('news', 0, 10);
```

Запись включается в результат, если:

- поле `status` содержит `publish`
- `publish_date` не задана или не превышает текущее время

Для публичных списков рекомендуется использовать этот метод вместо `getAllPost()`.

## Произвольная выборка

### getPostQuery($type, $query, $publish = false): array

Возвращает записи по переданному условию.

```php
$posts = Post::getPostQuery(
    'news',
    "`url` = 'release-notes' LIMIT 1",
    true
);
```

| Параметр | Описание |
|---|---|
| `$type` | Системное имя типа записи. |
| `$query` | SQL-условие без слова `WHERE`. |
| `$publish` | Добавляет проверку статуса и даты публикации. |

Получение одной опубликованной записи:

```php
$posts = Post::getPostQuery(
    'news',
    "`url` = 'release-notes' LIMIT 1",
    true
);

$post = $posts[1] ?? null;
```

Результат содержит `author_login` с логином автора.

Проверка публикации добавляется только для непустого `$query`.

## Объединение типов записей

### getPostQueryUnion($types, $where, $order = false, $limit = false): array

Выполняет общую выборку из нескольких типов записей.

Типы передаются строкой через запятую:

```php
$posts = Post::getPostQueryUnion(
    'news,articles',
    "`status` = 'publish'",
    '`publish_date` DESC',
    10
);
```

| Параметр | Описание |
|---|---|
| `$types` | Системные имена типов через запятую. |
| `$where` | Общее SQL-условие. |
| `$order` | Выражение сортировки. |
| `$limit` | Ограничение количества записей. |

Ключ `post_type` позволяет определить исходный тип каждой записи:

```php
foreach ($posts as $post) {
    echo Convert::sanitizeHtml($post['post_type']);
}
```

### getPublishPostUnion($types, $limit = '', $limit2 = '', $order = '`id`', $sort = 'DESC'): array

Возвращает опубликованные записи нескольких типов.

```php
$posts = Post::getPublishPostUnion(
    'news,articles',
    0,
    10,
    '`publish_date`',
    'DESC'
);
```

Метод проверяет статус и дату публикации каждой записи.

## Подсчёт записей

### postCount($type, $publish = false, $query = false): int|string

Возвращает количество записей указанного типа.

```php
$count = (int) Post::postCount('news');
```

Подсчёт опубликованных записей:

```php
$count = (int) Post::postCount('news', true);
```

Подсчёт по условию:

```php
$count = (int) Post::postCount(
    'news',
    true,
    "`author` = '5'"
);
```

### postCountUnion($types, $publish = false, $query = false): int

Возвращает общее количество записей нескольких типов.

```php
$count = Post::postCountUnion('news,articles', true);
```

## Добавление записи

### addPost($query, $type): string|false

Добавляет запись и возвращает её идентификатор.

```php
$postId = Post::addPost(
    "`title` = 'Новая запись', `url` = 'new-post', `status` = 'publish'",
    'news'
);
```

| Параметр | Описание |
|---|---|
| `$query` | SQL-выражение со значениями полей. |
| `$type` | Системное имя типа записи. |

Метод принимает готовый SQL-фрагмент. Не передавайте в него непроверенные пользовательские данные.

## Обновление записи

### updatePost($query, $id, $type): void

Обновляет поля записи.

```php
Post::updatePost(
    "`title` = 'Новый заголовок'",
    15,
    'news'
);
```

| Параметр | Описание |
|---|---|
| `$query` | SQL-выражение с изменяемыми полями. |
| `$id` | Идентификатор записи. |
| `$type` | Системное имя типа записи. |

Метод принимает готовый SQL-фрагмент. Не передавайте в него непроверенные пользовательские данные.

## Метаданные

### updateMeta($id, $type, $meta, $value): void

Добавляет или изменяет одно метаполе записи.

```php
Post::updateMeta(
    15,
    'news',
    'subtitle',
    'Краткое описание'
);
```

### updateMetaAll($id, $type, $meta): void

Полностью заменяет метаданные записи.

```php
Post::updateMetaAll(
    15,
    'news',
    [
        'subtitle' => 'Краткое описание',
        'featured' => 'on',
    ]
);
```

Существующие метаданные, которых нет в переданном массиве, будут удалены.

## Категории

### updateCategory($id, $type, $category, $value): void

Добавляет или изменяет значение одной категории записи.

```php
Post::updateCategory(
    15,
    'news',
    'important',
    '1'
);
```

### updateCategoryAll($id, $type, $category): void

Полностью заменяет локальные категории записи.

```php
Post::updateCategoryAll(
    15,
    'news',
    [
        'important' => '1',
        'events' => '0',
    ]
);
```

Существующие категории, которых нет в переданном массиве, будут удалены.

### getPublishPostCategory($type, $category, $limit = '', $limit2 = '', $order = '`id`', $sort = 'DESC'): array|null

Возвращает опубликованные записи, у которых в локальных категориях присутствует активное значение.

```php
$posts = Post::getPublishPostCategory(
    'news',
    'important',
    0,
    10
);
```

В текущей реализации параметр `$category` не участвует в условии выборки. Метод выбирает записи, содержащие значение категории `1`, независимо от её ключа.

В результате преобразуется только поле `meta`. Остальные структурированные поля остаются в исходном виде.

## Данные записи

Запись может содержать следующие ключи:

| Ключ | Описание |
|---|---|
| `id` | Идентификатор записи. |
| `title` | Заголовок. |
| `content` | Основное содержимое. |
| `url` | URL записи. |
| `img` | Путь к изображению. |
| `author` | Идентификатор автора. |
| `author_login` | Логин автора, если метод его добавляет. |
| `date` | Дата создания. |
| `publish_date` | Дата публикации. |
| `status` | Статус записи. |
| `meta` | Метаданные. |
| `category` | Локальные категории. |
| `global_meta` | Глобальные метаданные. |
| `global_category` | Глобальные категории. |
| `comment` | Комментарии. |
| `seo_title` | SEO-заголовок. |
| `seo_description` | SEO-описание. |
| `seo_key` | Ключевые слова. |
| `noindex` | Признак запрета индексации. |
| `canonical` | Канонический URL. |
| `more` | Дополнительное содержимое. |
| `views` | Количество просмотров. |
| `post_type` | Системное имя типа записи. |

## Важные замечания

- коллекции записей индексируются начиная с `1`
- для публичных списков используйте методы с проверкой публикации
- проверяйте существование записи перед обращением к её данным
- экранируйте текстовые значения перед выводом в HTML
- не передавайте пользовательские данные в SQL-фрагменты методов
- методы объединения принимают типы строкой через запятую
